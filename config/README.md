# Configuration Module

The `config` package centralizes all application settings.

Keeping configuration separate improves maintainability and simplifies deployment.

## Files

### settings.py

Stores paths, model names, dataset locations, and configurable parameters.

### validator.py

Validates required files and environment variables before application startup.

### version.py

Contains application metadata including version number and description.

## Purpose

- Centralized configuration
- Environment validation
- Version management
- Cleaner code organization
